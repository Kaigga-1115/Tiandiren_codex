# 網站資料說明

這份文件用來說明目前 `RED 傳說檔案網站` 的資料怎麼修改。

這個網站目前是 React 前端專案，資料先寫在前端裡，之後如果要接後端資料管理功能，也可以直接沿用這些資料欄位。

## 一、網站目前使用的資料類型

### 1. RED 基本資料

這一組資料是用來顯示 RED 的角色資訊，包含：

- 姓名
- 職稱
- 簡介
- 首次登場
- 所屬地區
- 戰鬥風格
- 代表稱號
- 相關作品
- 傳說紀錄

### 2. 寶可夢夥伴資料

這一組資料是用來顯示 RED 的寶可夢夥伴，包含：

- 寶可夢名稱
- 圖片
- 簡介
- 詳細介紹
- 性格特色
- 配招示意

### 3. 傳說紀錄資料

這一組資料是用來顯示 RED 的傳說檔案，包含：

- 欄位標題
- 欄位值
- 說明文字

## 二、資料放在哪裡修改

目前最主要的資料都整理在這個檔案：

- `frontend/src/data.js`

如果你要改文字、改圖片、增減資料，通常都先改這個檔案最方便。

## 三、要怎麼修改 RED 基本資料

如果你要改 RED 的介紹、稱號或其他角色資訊，請打開：

- `frontend/src/data.js`

裡面的這一段：

- `redProfile`

你可以直接修改：

- `name`：姓名
- `title`：代表稱號
- `intro`：首頁主視覺介紹文字
- `about`：關於 RED 的段落
- `facts`：角色資訊欄位

### 修改範例

如果你想把「戰鬥風格」改成別的說法，只要在 `facts` 裡找到對應欄位，修改 `value` 就可以。

例如：

```js
{
  label: "戰鬥風格",
  value: "沉著、直接、壓迫感強",
}
```

## 四、要怎麼修改寶可夢夥伴資料

寶可夢夥伴資料也在：

- `frontend/src/data.js`

裡面的這一段：

- `companions`

每一筆資料都代表一隻寶可夢夥伴。

你可以修改：

- `name`：寶可夢名稱
- `image`：寶可夢圖片
- `intro`：卡片上的簡短介紹
- `detail`：右側詳情面板的完整介紹
- `nature`：性格特色
- `moves`：配招示意

### 修改方式

如果你要把某隻寶可夢換掉，只要改那一筆資料即可，不需要動整個網頁。

例如你想把皮卡丘改成另一隻寶可夢，就把那一筆的：

- 名稱改掉
- 圖片改掉
- 介紹文字改掉

這樣頁面就會自動更新。

## 五、要怎麼修改傳說紀錄資料

傳說紀錄資料也在：

- `frontend/src/data.js`

裡面的這一段：

- `records`

每一筆資料都有：

- `title`：欄位標題
- `value`：欄位值
- `text`：說明文字
- `accent`：檔案編號或標記

### 修改方式

如果你想把「白銀山最終對戰」改成別的內容，可以直接改那一筆資料，不需要去找 JSX 版面。

## 六、圖片要放在哪裡

目前圖片資料夾請放在 `frontend/` 底下。

### RED 圖片

請放在：

- `frontend/角色圖片`

這裡目前放的是 RED 的不同角色圖，像是：

- `Red-FRLG.png`
- `Red-LetsGo.png`
- `Red-SunMoon.png`
- `Red-Computer.png`
- `Red-Ariga.jpg`

### 寶可夢圖片

請放在：

- `frontend/經歷圖片`

這裡目前放的是寶可夢夥伴的圖片，例如：

- `Pikachu.png`
- `Snorlax.png`
- `Venusaur.png`
- `Charizard.png`
- `Blastoise.png`
- `Espeon.png`

## 七、圖片名稱建議怎麼命名

圖片名稱建議用簡單、清楚、好辨認的方式命名。

### 建議格式

- 角色圖：`Red-FRLG.png`
- 寶可夢圖：`Pikachu.png`
- 如果有同一角色不同版本，可以加版本名

例如：

- `Red-LetsGo.png`
- `Red-SunMoon.png`
- `Pikachu-alt.png`

### 命名原則

- 盡量用英文或拼音
- 不要太長
- 不要加太多奇怪符號
- 同一類圖片盡量保持命名風格一致

## 八、如果你要讓網站更好維護，建議怎麼改

目前網站資料已經整理成比較好改的形式：

- `frontend/src/data.js` 負責資料
- `frontend/src/App.jsx` 負責畫面結構
- `frontend/src/styles.css` 負責外觀與動畫

這樣的好處是：

- 想改字，只要改資料檔
- 想改圖片，只要換圖片檔
- 想改動畫，只要改樣式檔

對初學者來說，會比把所有內容直接寫在元件裡更好維護。

## 九、最常用的修改方式

### 改文字

到 `frontend/src/data.js` 改。

### 改圖片

把新圖片放到對應資料夾，再到 `frontend/src/data.js` 改檔名。

### 改版面

到 `frontend/src/App.jsx` 或 `frontend/src/styles.css` 改。

## 十、這個專案目前的整理方式

現在網站資料已經被分成三大部分：

- `redProfile`
- `companions`
- `records`

這樣未來如果要接後端資料，會很容易把它們改成 API 資料來源。
